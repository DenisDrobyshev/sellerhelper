## Проблема

Большинство тех, кто хочет выйти на маркетплейс, спотыкаются на нулевом шаге: **выбирают не тот товар.** Они либо:

- смотрят на пустой шаблон бизнес-плана — и бросают, либо
- жмут кнопку «AI-генератор бизнес-плана» и получают красивый PDF с выдуманными цифрами.

И то, и другое поощряет **фейковый прогресс** (симпатичный план) вместо **реального** — доказательства, что кто-то действительно купит.

А инструменты, у которых _есть_ реальные данные (MPStats, Moneyplace, Jungle Scout, Helium 10), — это **дашборды**: вываливают на тебя цифры и предполагают, что ты уже знаешь, что с ними делать. Ни один не ведёт новичка за руку.

## Чем SellerCompass отличается

SellerCompass — это **не дашборд и не генератор плана**. Это **пошаговый pipeline со стадиями-воротами**, который ведёт новичка от идеи до проверенного первого товара — и **не пускает дальше, пока шаг не подтверждён реальными данными.**

1. **Доказательства вместо мнений.** Каждая рекомендация опирается на живые данные маркетплейса, а не на фантазию LLM.
2. **Ворота, а не пустой холст.** Нельзя перепрыгнуть к «заказывай партию», пока не пройдены ворота спроса и юнит-экономики.
3. **Сооснователь, а не отчёт.** AI объясняет _почему_, подсвечивает риски и говорит **pivot или kill**, когда так показывают данные.

## 5 стадий (v0 — Wildberries)

| # | Стадия | Что происходит | Ворота |
|---|--------|----------------|--------|
| 1 | **Поиск ниши** | Бюджет + интересы + цель → кандидатные ниши | — |
| 2 | **Проверка спроса** | Реальные запросы, продажи топ-игроков, тренд | Спрос выше порога и не падает |
| 3 | **Оценка конкуренции** | Плотность, ценовые коридоры; NLP по отзывам конкурентов → незакрытые боли | Есть окно для входа |
| 4 | **Юнит-экономика** | Закуп, логистика, комиссии, реклама, цена → маржа, точка безубыточности, бюджет | Маржа положительная и реалистичная |
| 5 | **Решение** | Сводный вердикт **Go / Pivot / Kill** + план первой партии + чек-лист запуска | — |

Полное описание каждых ворот: [METHODOLOGY.md](METHODOLOGY.md).

## Open-core

- **Открыто (этот репозиторий, MIT):** движок методологии, коннекторы к маркетплейсам и локальная self-host версия — со своим ключом от LLM.
- **Облако (в планах):** уже собранные живые **и исторические** данные, без настройки. **Ров — это инфраструктура данных**, а не алгоритм.

## Быстрый старт

```bash
git clone https://github.com/DenisDrobyshev/sellercompass
cd sellercompass
cp .env.example .env          # добавь ключ LLM; при необходимости укажи WB_PROXY_URL

# Вариант A — Docker (API + Postgres + Redis)
docker compose up --build     # -> http://localhost:8000/docs

# Вариант B — локальный Python
pip install -e ".[dev]"
uvicorn core.main:app --reload

# Смоук-тест коллектора Wildberries
python -m core.collectors.wildberries "чехол для iphone"
```

> **Важно про данные.** Wildberries режет запросы с IP дата-центров ошибкой HTTP 429. Запускай коллектор из резидентной / RU-сети или задай `WB_PROXY_URL`. Это ключевая сложная часть продукта — коллектор решает её через экспоненциальный backoff, вежливый throttling и поддержку прокси.

## Структура проекта

```
core/
  api/          маршруты FastAPI
  collectors/   коннекторы к маркетплейсам (Wildberries)
  engine/       стейт-машина стадий-ворот
  models/       нормализованные модели данных
tests/          юнит-тесты (CI: ruff + pytest)
```

## Стек

Python · FastAPI · PostgreSQL · Redis · async-коллекторы (httpx) · Docker Compose. В планах: LLM-оркестрация + RAG по рыночным данным, и ML — прогноз спроса, скоринг ниш, NLP по отзывам. См. [ARCHITECTURE.md](ARCHITECTURE.md).
